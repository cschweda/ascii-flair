# ascii-flair

Minimal ASCII art and styled text for `console.log` — works everywhere.

- **~1-2KB** core (text mode)
- **~1-2KB** per font (lazy-loaded on demand)
- **Zero runtime dependencies** — fonts are pre-compiled
- Works in **browser** and **terminal**
- Framework-agnostic: Vue, React, Angular, Nuxt, Next, plain Node.js

## Install

```bash
npm install ascii-flair
```

## Usage

```js
import { flair } from 'ascii-flair'

// ASCII art mode
flair('My App', 'flair', 'standard')

// ASCII art with styling
flair('My App', 'flair', 'doom', { color: 'cyan', border: true })

// Styled plain text
flair('Server started on :3000', 'text', { color: 'green', bold: true })

// Plain text with border
flair('v2.0', 'text', { border: true, padding: 1 })
```

## API

```
flair(text, mode, fontOrOptions?, options?)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `text` | string | The text to render |
| `mode` | `'flair'` \| `'text'` | ASCII art or styled plain text |
| `fontOrOptions` | string \| object | Font name (flair mode) or options (text mode) |
| `options` | object | Styling options (flair mode) |

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `color` | string | — | `cyan`, `green`, `red`, `yellow`, `blue`, `magenta`, `white` |
| `bold` | boolean | `false` | Bold text (terminal only) |
| `border` | boolean | `false` | Wrap in a box border |
| `padding` | number | `0` | Lines of padding around text |

> **Note:** `flair()` returns a Promise in `'flair'` mode (fonts are lazy-loaded). In `'text'` mode it is synchronous.

## Development

### Setup

```bash
git clone https://github.com/cschweda/ascii-flair.git
cd ascii-flair
npm install
```

### Dev Workflow

```bash
# Run tests in watch mode (re-runs on file changes)
npm run test:watch

# Run tests once
npm test

# Compile fonts from figlet (only needed if changing the font list)
npm run build:fonts

# Regenerate README font gallery
npm run build:readme

# Build the dist/ output (ESM + CJS)
npm run build

# Full build pipeline (fonts + readme + dist)
npm run build:fonts && npm run build:readme && npm run build
```

### Making Changes

1. Edit files in `src/` (the source code)
2. Run `npm run test:watch` to verify your changes
3. Run `npm run build` to generate the `dist/` output
4. Test the built module: `node -e "import('./dist/ascii-flair.es.js').then(m => m.flair('Test', 'flair', 'doom'))"`

### Project Structure

```
src/
  index.js      Main flair() API
  detect.js     Browser vs terminal detection
  renderer.js   Minimal ASCII art renderer
  styles.js     ANSI colors, borders, padding, browser CSS
  fonts/        Pre-compiled font data (generated by build:fonts)
scripts/
  compile-fonts.js    Extracts fonts from figlet into compact JS modules
  generate-readme.js  Renders font gallery for this README
tests/            Test files (vitest)
dist/             Built output (generated by build)
```

### Publishing

**Automatic (via GitHub Actions):**

Tag a release and the CI will publish to npm:

```bash
npm version patch   # or minor / major
git push origin main --tags
```

The `publish.yml` workflow triggers on `v*` tags, runs tests, builds, and publishes.

> **Setup required:** Add your npm token as a GitHub secret named `NPM_TOKEN`.
> Generate one at https://www.npmjs.com/settings/~/tokens (choose "Automation" type).

**Manual:**

```bash
npm run release:patch   # bump version + publish
npm run release:minor
npm run release:major
```
